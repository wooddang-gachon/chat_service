# 카카오톡 분석 및 AI 에이전트 기반 DB 설계 계획서

이 문서는 카카오톡의 방대한 서비스를 분석하고, 이를 데이터베이스(DB)로 설계하기 위해 AI 에이전트를 구축 및 활용하여 문제에 접근하는 프로세스를 정의합니다.

## Goal Description
- **목적**: 카카오톡의 주요 기능군을 식별하고, 해당 기능을 지원할 수 있는 확장성 있고 효율적인 데이터베이스 구조를 도출합니다. 이 과정을 AI 에이전트들을 설계하고 협업하도록 하여 자동화 및 정밀화합니다.
- **접근 전략**: 수동으로 설계를 시작하기에 앞서, 문제에 체계적으로 접근하는 단계별 아키텍처와 방법론을 먼저 수립합니다.

## 문제 접근 과정 (Problem Approach Process)

### 1단계: 카카오톡 서비스 기능 역공학 및 카테고리화
- **목표**: 카카오톡이 제공하는 모든 기능적 도메인을 분류합니다.
- **주요 도메인 예시**:
  - 회원 및 프로필 (친구 관계, 상태 메시지, 멀티프로필)
  - 채팅 (1:1 채팅, 그룹 채팅, 오픈채팅, 비밀채팅)
  - 메시지 전송 및 관리 (텍스트, 멀티미디어, 읽음 확인, 멘션, 답장, 전달)
  - 부가 기능 (선물하기, 쇼핑, 채널, 페이, 캘린더 등)

### 2단계: AI 에이전트 아키텍처 설계
- **목표**: DB 설계를 단계별로 자동화 및 검증할 멀티 에이전트 시스템을 구상합니다.
- **예상 에이전트 라인업**:
  - **도메인 분석가 에이전트 (Domain Analyst Agent)**: 비즈니스 요구사항을 분석하고 핵심 엔티티(Entity) 및 비즈니스 규칙 추출
  - **데이터베이스 모델러 에이전트 (DB Modeler Agent)**: 추출된 엔티티를 바탕으로 관계형/비관계형 데이터 모델(스키마) 설계 및 SQL/NoSQL 구조 정의
  - **최적화 및 보안 검증 에이전트 (Optimizer & Security Validator Agent)**: 정규화 레벨 검토, 역정규화 필요성 판단, 인덱싱 전략 수립 및 보안 요구사항(개인정보 암호화 등) 검증

### 3단계: 점진적 DB 모델링 프로세스 수행
- **과정**:
  1. 도메인 분석가 에이전트가 특정 도메인(예: 채팅방 및 메시지 전송)의 스펙을 작성합니다.
  2. 데이터베이스 모델러 에이전트가 해당 스펙을 입력받아 ERD(Entity Relationship Diagram) 구조와 DDL을 도출합니다.
  3. 검증 에이전트가 설계된 스키마에 대해 동시성 제어, 트래픽 분산(샤딩/파티셔닝), 조회 쿼리 최적화 관점에서 피드백을 제공합니다.
  4. 최종 결과를 `plan.md` 및 하위 설계 문서에 아티팩트로 누적합니다.
