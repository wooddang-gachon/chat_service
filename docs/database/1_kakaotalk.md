# 카카오톡 기능 분석서 (1_kakaotalk.md)

이 문서는 데이터베이스 설계를 위해 카카오톡(KakaoTalk) 서비스의 주요 도메인별 기능적 요구사항과 비즈니스 룰을 역공학하여 명세한 문서입니다.

---

## 1. 회원 및 프로필 도메인 (User & Profile)

### 1.1 계정 및 가입 정보
- **계정 식별**: 고유 유저 ID, 전화번호(국가코드 포함, 1인 1계정 원칙), 이메일 주소(카카오계정 연동).
- **상태**: 활성화, 탈퇴 대기, 차단(영구/기간제).

### 1.2 프로필 관리
- **기본 프로필**: 닉네임, 프로필 이미지(원본/썸네일), 배경 이미지, 상태 메시지, 프로필 내 스티커 및 위젯(디데이, 배경음악 등).
- **멀티프로필**: 
  - 기본 프로필 외에 최대 3개까지 생성 가능.
  - 특정 친구를 지정하여 해당 친구에게만 멀티프로필을 보여주도록 매핑.
  - 친구 목록 관점에서 노출되는 프로필 정보를 동적으로 결정해야 함.

### 1.3 친구 관계 (Friendship)
- **관계 유형**:
  - 일반 친구: 상대방의 전화번호를 저장하거나 카카오톡 ID로 추가한 상태.
  - 즐겨찾기(별표): 상단에 우선 노출할 친구.
  - 숨김 친구: 친구 목록에서 보이지 않지만, 메시지는 수신 가능.
  - 차단 친구: 메시지 수신 차단, 혹은 메시지 수신 및 프로필 비공개 차단.
- **추천 친구**: 내 연락처에 있으나 등록되지 않은 유저, 알 수도 있는 친구 알고리즘 노출 유저.

### 1.4 친구 목록 화면 구성 및 노출 그룹화
- **섹션별 노출 그룹**:
  - **내 프로필**: 내 기본 프로필 정보 및 내 멀티프로필 목록(나에게만 노출).
  - **생일인 친구**: 생일 당일인 친구 목록(오늘 생일, 선물하기 바로가기 연동) 및 생일이 지난 친구/다가오는 생일 친구 목록(옵션 설정 가능).
  - **업데이트한 프로필**: 최근(예: 3일 이내) 프로필 사진, 배경 사진, 상태 메시지 등을 변경한 친구 목록 모아보기.
  - **즐겨찾기(별표)**: 사용자가 즐겨찾기 등록한 친구들을 최상단에 우선 배치.
  - **추천 친구**: 아직 친구로 등록하지 않았으나 내 기기 연락처와 동기화되어 추천된 친구, 또는 나와 매핑될 수 있는 잠재적 친구.
  - **채널/비즈니스**: 내가 추가한 공식 채널 및 비즈니스 계정 목록 분리 노출.
  - **일반 친구 목록**: 가나다순/알파벳순으로 정렬된 일반 친구 목록.
- **정렬 및 노출 배제 정책**:
  - 친구 목록은 '내 프로필 -> 즐겨찾기 -> 생일인 친구 -> 업데이트한 친구 -> 채널 -> 일반 친구' 순으로 정렬되어 노출됨.
  - 숨김 처리된 친구나 차단 처리된 친구는 메인 '친구 목록' 화면의 모든 섹션에서 완전히 제외되어야 함 (별도의 숨김/차단 친구 관리 메뉴에서만 조회 가능).
  - 이름 검색(실시간 텍스트 필터링) 기능이 제공되어야 함.

## 2. 채팅방 도메인 (Chat Room)

### 2.1 채팅방 유형 및 특징
- **나와의 채팅방**: 1인 메모용 채팅방. 나 이외의 참여자 없음.
- **1:1 채팅방**: 2명의 유저 간 개설. 상대방이 방을 나가더라도 대화 상대방 정보는 유지되며, 새로운 메시지가 오면 방이 다시 활성화됨.
- **그룹 채팅방**: 3인 이상의 일반 채팅방. 참여자가 자유롭게 링크로 초대하거나 직접 친구를 초대 가능.
- **오픈채팅방**:
  - 실명 프로필이 아닌 '오픈프로필' 혹은 카카오 프렌즈 프로필로 참여 가능.
  - **오픈채팅 1:1 방**: 링크 생성자가 존재하고 참여자가 들어오는 일대일 구조.
  - **오픈채팅 그룹 방**: 다수가 참여 가능. 방장(Owner)과 부방장(Manager) 권한이 존재하며 강퇴, 공지 설정 기능이 있음.
  - **접근 제어**: 비밀번호(참여 코드) 설정 가능.

### 2.2 채팅방 메타데이터 및 개인화 설정
- **방 정보**: 방 이름(유저별 개별 변경 가능 또는 참여자 이름 자동 조합), 방 프로필 이미지.
- **사용자별 개인화**: 알림 설정(On/Off), 상단 고정 여부(최대 고정 개수 제한 존재), 배경화면 커스텀 설정, 입력창 잠금 설정.
- **세션 상태**: 마지막 읽은 메시지 ID (안 읽은 메시지 개수 계산용), 채팅방 참여 시점 및 퇴장 시점 기록.

## 3. 메시지 전송 및 상호작용 도메인 (Message & Interaction)

### 3.1 메시지 유형
- **텍스트 메시지**: 글자 수 제한이 있는 일반 텍스트 데이터.
- **미디어 메시지**: 이모티콘(공식 기본/미니/구매 이모티콘), 사진/GIF, 동영상, 음성 필터 메시지, 파일.
- **기타 특수 메시지**: 지도(위치 정보), 연락처 공유, 일정 공유.
- **시스템 메시지**: "XXX님이 들어왔습니다.", "XXX님이 나갔습니다." 등.

### 3.2 메시지 상태 및 편집 관리
- **안 읽은 카운트**: 채팅방 참여자 수에서 해당 메시지를 읽은 사람의 수를 뺀 값.
- **메시지 삭제**:
  - 나에게만 삭제: 내 기기에서만 메시지를 숨김.
  - 모든 대화 상대에게서 삭제: 발송 후 5분 이내의 메시지만 가능하며, 삭제 시 "삭제된 메시지입니다."로 내용이 변경됨.
- **메시지 수정**:
  - 텍스트 타입 메시지만 수정 가능.
  - **제한 조건**: 전송 완료 후 5분 이내에만 수정할 수 있음.
  - **상태 마킹**: 수정된 메시지는 UI에 "편집됨" 또는 "수정됨" 플래그를 표기해야 함.
  - **데이터베이스 반영점**: 메시지 테이블에 `is_edited(Boolean)`, `last_edited_at(Datetime)` 컬럼이 요구되며, 수정 이력(History) 기록 여부는 보안 및 정합성 정책에 따름.

### 3.3 메시지 부가 기능
- **답장(Reply)**: 특정 이전 메시지 ID를 부모(Parent Message)로 지정하여 연결하는 스레드형 구조.
- **공지사항(Notice)**: 채팅방 상단에 고정하는 메시지. 방당 1개 혹은 다수 등록 가능.
- **멘션(Mention)**: `@닉네임` 형태로 특정 대화 상대를 지정하여 알림 유발.

## 4. 이모티콘 및 이모지 시스템 (Emoticon & Emoji System)

### 4.1 이모티콘 종류 및 속성
- **이모티콘 대분류**:
  - **공식 기본 이모티콘 (Official Default)**:
    - 앱 설치 시 무료로 기본 제공되는 카카오프렌즈(어피치, 라이언 등) 이모티콘 세트.
    - 별도의 라이선스 구매나 검증 없이 모든 사용자가 영구 무료 사용 가능.
  - **구매형 일반 이모티콘 (Store Emoticon)**:
    - 이모티콘 스토어에서 판매되는 상품. 정적/움직이는 이모티콘, 사운드형 등이 존재함.
  - **미니 이모티콘 (Mini Emoticon)**:
    - 텍스트 메시지 입력창 내부에서 글자(Text)와 함께 줄 바꿈 없이 혼합하여 인라인(Inline)으로 사용할 수 있는 아주 작은 크기의 캐릭터 이모티콘.
    - 단독 전송 시 일반 이모티콘처럼 크게 출력되기도 함.
    - **DB 설계 고려사항**: 메시지 테이블의 본문 텍스트 내에 미니 이모티콘 매핑 코드(예: `[mini_emoticon_id:123]`)를 치환하여 저장하거나, 메시지-미니이모티콘 매핑 관계 테이블을 두어야 정상적인 렌더링이 가능함.
  - **카카오 기본 텍스트 이모티콘 (괄호 이모티콘 - 신규 피드백 반영)**:
    - 사용자가 텍스트 입력창에 특정 텍스트 패턴(예: `(미소)`, `(하트)`, `(눈물)`, `(화남)` 등)을 입력하면, 화면상에서 둥근 노란색 얼굴 그래픽(카카오 기본 이모지)으로 변환되어 텍스트 사이에 작은 크기로 표시되는 기능.
    - **DB 설계 고려사항**: 실제 DB에는 사용자가 입력한 순수 텍스트(예: `(미소) 안녕하세요`) 형태 그대로 저장되며, 클라이언트 렌더링 시 매핑 규칙에 따라 이미지 파일로 치환됨. 따라서 서버에서는 `텍스트-이미지 매핑 테이블`이 요구됨.
  - **OS 유니코드 이모지 (Unicode Emoji - 신규 피드백 반영)**:
    - iOS, Android 등 모바일 OS의 가상 키보드에서 제공하는 국제 표준 이모티콘 캐릭터(😀, 😂, ❤️ 등).
    - **DB 설계 고려사항**: 데이터베이스 캐릭터 셋(Character Set)이 이모지 캐릭터(4바이트 문자)를 완벽히 지원할 수 있도록 반드시 **`utf8mb4`** 인코딩으로 설계되어야 함. 그렇지 않을 경우 데이터 저장 시 DB 에러가 발생하거나 텍스트가 유실됨.
- **유형 분류**:
  - **멈춰있는 이모티콘**: 정적 이미지(PNG, JPG 등).
  - **움직이는 이모티콘 (Animated)**: 동적 루프 이미지(GIF, WebP 등) 및 재생 제어 플래그.
  - **사운드 이모티콘**: 소리 오디오 파일 재생 기능이 연결된 움직이는 이모티콘.
  - **큰 이모티콘**: 채팅방 레이아웃 상 크게 표현되거나 오버레이 효과가 적용되는 특수 규격.

### 4.2 라이선스 및 사용 권한
- **단품 구매형 라이선스**:
  - 유저가 특정 이모티콘 패키지를 구매하여 영구적 혹은 기간 한정으로 사용할 권한을 획득함.
  - 유저 ID, 패키지 ID, 획득일, 만료일 매핑 테이블 필요.
- **구독형 라이선스 (이모티콘 플러스)**:
  - 매월 정기 결제를 수행하는 유저는 전체 이모티콘 풀 또는 구독 전용 이모티콘 팩을 무료로 제한 없이 다운로드하고 사용할 수 있음.
  - 유저 구독 상태 테이블 (구독 상태, 결제일, 다음 결제일, 만료일 등) 및 구독을 통한 이모티콘 팩 임시 획득 로그 기록.
  - 채팅 입력창에서 단어를 입력하면 연관된 이모티콘을 추천(이모티콘 자동 제안)하는 태그 기반 매핑 정보 필요.

## 5. 부가 기능 및 미디어 관리 (Utilities & Storage)

### 5.1 톡캘린더 (Calendar)
- 채팅방 참여자들과 일정을 공유하고 알림을 설정하는 기능.
- 일정 제목, 시작/종료 시간, 위치, 참석 대상자, 채팅방 연동 정보 포함.

### 5.2 미디어 저장소 및 만료 정책
- 사진, 동영상, 파일 등 대용량 데이터는 원본 서버 저장 기한(만료일)이 존재함. 만료일이 지나면 썸네일만 표시되고 원본 다운로드는 불가.
- 톡서랍(유료 구독 서비스) 이용자는 만료일 없이 평생 보관 가능.
